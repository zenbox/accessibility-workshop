ARIA und Landmarks

ARIA steht für Accessible Rich Internet Applications und ist eine Spezifikation des W3C, die es ermöglicht, Webseiten und Webanwendungen für Menschen mit Behinderungen, insbesondere für die Interpretation durch Screenreader zugänglicher zu machen.

ARIA enthält mehrere Hauptkomponenten:

  • Properties: Attribute, die
    1. Textinformationen hinzufügen von Elementen beschreiben.
    2. States: Eine weitere Gruppe umfasst das Verhalten bei Änderungen des DOM.
    3. Schließlich gibt es noch Relationen, die Beziehungen zwischen Elementen beschreiben.
  • Roles: Rollen, die Elementen zugewiesen werden können, um die semantische Bedeutung eines Elements zu erzeugen oder zu ändern.

Die folgenden Beispiele sind nicht vollständig. Eine vollständige Beschreibung aller ARIA-Attribute gibt es zum Beispiel bei SELFHTML.

ARIA-Attribute, die Textinformationen für Screenreader hinzufügen

Die folgenden Attribute können Textinformationen für Screenreader hinzufügen:

aria-label
Ein Label, das den Inhalt eines Elements beschreibt. Es wird von Screenreadern vorgelesen, wenn das Element fokussiert ist. <button aria-label="Schließen" class="fa fa-close"></button>
aria-labelledby
Die ID eines anderen Elements, dessen Inhalt als Label für das aktuelle Element verwendet wird. <span id="dialog-title">Fenster schließen</span>
<button aria-labelledby="dialog-title" type="button">×</button>
aria-describedby
Die ID eines anderen Elements, dessen Inhalt als Beschreibung für das aktuelle Element verwendet wird. <p class="sr-only" id="close-help">Schließt das Fenster.</p>
<button type="button" aria-describedby="close-help">Schließen</button>

ARIA-Attribute, die den Zustand von Elementen beschreiben

Die folgenden Attribute können den Zustand von Elementen beschreiben:

aria-disabled
Gibt an, ob ein Element deaktiviert ist. <div role="button" aria-disabled="true" tabindex="0">Speichern</div>
aria-hidden
Gibt an, ob ein Element für Screenreader sichtbar ist. <div aria-hidden="true">
aria-checked
Gibt an, ob ein Element ausgewählt ist. <div role="checkbox" aria-checked="true" tabindex="0">Newsletter</div>
aria-pressed
Gibt an, ob ein Element gedrückt ist. <button aria-pressed="true">
aria-expanded
Gibt an, ob ein Element erweitert ist. <button aria-expanded="true" aria-controls="details" type="button">Details</button>
<div id="details">…</div>
aria-selected
Gibt an, ob ein Element innerhalb eines Auswahl-Widgets ausgewählt ist (z.B. in listbox, tabs). <div role="tablist" aria-label="Ansichten">
  <button role="tab" aria-selected="true" aria-controls="panel-a" id="tab-a" type="button">Übersicht</button>
  <button role="tab" aria-selected="false" aria-controls="panel-b" id="tab-b" type="button">Details</button>
</div>
<section role="tabpanel" id="panel-a" aria-labelledby="tab-a">…</section>
<section role="tabpanel" id="panel-b" aria-labelledby="tab-b" hidden>…</section>
aria-current
Markiert das „aktuelle“ Element in einer Menge verwandter Elemente (z.B. aktuelle Seite, aktueller Schritt). <nav aria-label="Brotkrumen">
  <ol>
    <li><a href="/">Start</a></li>
    <li><a href="/konto" aria-current="page">Konto</a></li>
  </ol>
</nav>
aria-required
Kennzeichnet ein Eingabefeld als erforderlich. (Hinweis: In HTML ist required oft die bessere Wahl.) <label for="email">E‑Mail</label>
<input id="email" name="email" type="email" aria-required="true" autocomplete="email">
aria-invalid
Markiert ein Feld als ungültig. <label for="zip">PLZ</label>
<input id="zip" name="zip" inputmode="numeric" aria-invalid="true" aria-describedby="zip-error">
<p id="zip-error" class="error">Bitte eine gültige PLZ eingeben.</p>
aria-errormessage
Verknüpft ein ungültiges Feld mit einem Element, das die Fehlermeldung enthält. Wird typischerweise zusammen mit aria-invalid="true" eingesetzt. <label for="password">Passwort</label>
<input id="password" name="password" type="password" aria-invalid="true" aria-errormessage="pw-error">
<p id="pw-error" class="error">Mindestens 12 Zeichen.</p>
aria-modal
Kennzeichnet einen Dialog als modal, d.h. der Fokus bleibt im Dialog, bis er geschlossen wird. <div role="dialog" aria-modal="true" aria-labelledby="dialog-title">…</div>

und so weiter.

ARIA-Attribute, die beobachten, ob es am DOM Veränderungen gibt:

Die folgenden Attribute können beobachten, ob es am DOM Veränderungen gibt:

aria-atomic
Gibt an, ob ein Element atomar ist, d.h. ob es als Ganzes gelesen werden soll. <div role="alert" aria-atomic="true">
<p>Wichtige Nachricht:</p>
<p>Ihr Profil wurde erfolgreich aktualisiert.</p>
</div>
aria-busy
Gibt an, ob ein Element gerade beschäftigt ist. <section aria-busy="true" aria-label="Suchergebnisse">…</section>
aria-live
Gibt an, ob Änderungen an einem Element sofort vorgelesen werden sollen. <div role="status" aria-live="polite" aria-atomic="true">
  0 Treffer
</div>
aria-relevant
Gibt an, welche Art von Änderungen an einem Element vorgelesen werden sollen. <div role="status" aria-live="polite" aria-relevant="additions">
  …
</div>

ARIA-Attribute, die ein Verhalten von Elementen beschreiben

Die folgenden Attribute können Verhalten/Beziehungen von Elementen beschreiben:

aria-haspopup
Gibt an, ob ein Element ein Popup (z.B. menu, listbox oder dialog) öffnet. <button aria-haspopup="menu" aria-expanded="false" aria-controls="user-menu" type="button">Profil</button>
<div id="user-menu" role="menu">…</div>
aria-owns
Die ID eines anderen Elements, das von diesem Element besessen wird. <div role="listbox" aria-owns="opt-1 opt-2">…</div>
<div id="opt-1" role="option">Option 1</div>
<div id="opt-2" role="option">Option 2</div>
aria-controls
Die ID eines anderen Elements, das von diesem Element kontrolliert wird. <button aria-controls="filters" aria-expanded="false" type="button">Filter</button>
<section id="filters">…</section>
aria-flowto
Die ID eines anderen Elements, das von diesem Element fokussiert wird. <p id="step-1" aria-flowto="step-2">Schritt 1…</p>
<p id="step-2">Schritt 2…</p>
aria-activedescendant
Die ID eines anderen Elements, das von diesem Element aktiviert wird. <div role="listbox" tabindex="0" aria-activedescendant="opt-2">…</div>
<div id="opt-1" role="option">Option 1</div>
<div id="opt-2" role="option" aria-selected="true">Option 2</div>
aria-valuenow
Der aktuelle numerische Wert eines Widgets (z.B. slider). <div role="slider" tabindex="0" aria-label="Lautstärke" aria-valuemin="0" aria-valuemax="100" aria-valuenow="30">…</div>
aria-valuemin, aria-valuemax
Untere und obere Grenze eines numerischen Widgets. <div role="spinbutton" tabindex="0" aria-label="Anzahl" aria-valuemin="1" aria-valuemax="10" aria-valuenow="1">1</div>
aria-valuetext
Textuelle Beschreibung des aktuellen Werts (wenn aria-valuenow alleine nicht ausreicht). <div role="slider" tabindex="0" aria-label="Zoom" aria-valuemin="0" aria-valuemax="4" aria-valuenow="2" aria-valuetext="200%">…</div>

und so weiter.

Landmarks (roles)

Landmarks sind semantische Rollen, die es Screenreadern ermöglichen, die Struktur einer Webseite zu erkennen. Sie können nicht-semantischen div-Elemente eine semantische Bedeutung geben. Sie können auch die Semantik eines HTML-Elements ändern.

In HTML gilt: Wenn ein passendes semantisches Element existiert (<header>, <nav>, <main>, <aside>, <footer>), ist das meistens die bessere Wahl als ein zusätzliches role auf einem <div>.

Wenn es mehrere Bereiche mit derselben Landmark-Rolle gibt (z.B. mehrere Navigationsbereiche), sollten sie eindeutig benannt werden (z.B. mit aria-label oder aria-labelledby), damit Screenreader sie gut unterscheiden können.

role="banner"
Ein Bereich, der den Kopf einer Webseite enthält. Er kann mehrere Elemente enthalten, wie z.B. ein Logo, eine Überschrift, eine Suchfunktion und eine Navigation. <header role="banner">…</header>
role="navigation"
Ein Bereich, der eine Navigation enthält. Er kann mehrere Elemente enthalten, wie z.B. eine Liste von Links oder eine Suchfunktion. <nav aria-label="Hauptnavigation">…</nav>
role="main"
Der Hauptinhalt einer Webseite. Er kann mehrere Elemente enthalten, wie z.B. Artikel, Abschnitte und Nebeninhalte. <main>…</main>
role="complementary"
Ein Bereich, der zusätzliche Informationen enthält, die nicht zum Hauptinhalt gehören. Er kann mehrere Elemente enthalten, wie z.B. eine Sidebar, eine Werbung oder verwandte Links. <aside aria-label="Weiterführende Links">…</aside>
role="contentinfo"
Ein Bereich, der Informationen über die Webseite enthält. Er kann mehrere Elemente enthalten, wie z.B. ein Impressum, eine Datenschutzerklärung und eine Kontaktadresse. <footer>…</footer>
role="search"
Ein Bereich, der eine Suchfunktion enthält. Er kann mehrere Elemente enthalten, wie z.B. ein Eingabefeld, eine Schaltfläche und eine Ergebnisliste. <form role="search" aria-label="Website durchsuchen">…</form>
role="form"
Ein Bereich, der ein Formular enthält. Er kann mehrere Elemente enthalten, wie z.B. Eingabefelder, Schaltflächen und Auswahllisten. <form role="form" aria-labelledby="contact-title">…</form>
role="region"
Ein wichtiger Abschnitt der Seite, der als Landmark auftauchen soll. Regionen sollten immer benannt werden (z.B. per aria-label), sonst ist der Landmark-Eintrag für Nutzer:innen wenig hilfreich. <section role="region" aria-label="Aktuelle Hinweise">…</section>

und so weiter.
