Diese Simulation zeigt, wie ein Screenreader Inhalte vorliest und navigiert. Die Steuerelemente befinden sich im Overlay oben rechts. Verwenden Sie die Tastaturkürzel oder die virtuelle Tastatur, um zu navigieren. Es ist eine Simulation und kann vom Original abweichen.

Screenreader-Simulation

Der Screenreader muss neben der Tastatursteuerung, von der er übrigens das meiste selbst mitbringt, alle textbasierten Informationen korrekt vorlesen. Dazu gehören auch die ARIA-Attribute, die dem Screenreader zusätzliche Informationen geben, wenn das, was HTML beinhaltet, nicht ausreicht.

Auf dieser Seite sind Beispiele für Screenreader-Verhalten. Es wird gezeigt, wie eine DOM-Veränderung vorgelesen wird, wie Abkürzungen und fremde Sprachen funktionieren und wie eine Drag and Drop Tastatur-Bedienung funktioniert und vorgelesen wird .

Abschnitt 1: Einführung

Willkommen auf unserer Beispiel-Webseite. Hier können Sie die Funktionsweise eines Screenreaders testen.

Mehrsprachige Beispiele

Nachfolgend sehen Sie einige Beispiele für die Spracherkennung des Screenreaders:

Deutschen Text (Standard)

Dies ist ein deutscher Text, der in der Standardsprache vorgelesen wird.

Englischer Text

This is an English text that will be read using an English voice if available.

Französischer Text

Ceci est un texte français qui sera lu avec une voix française si disponible.

Spanischer Text

Este es un texto en español que se leerá con una voz en español si está disponible.

Gemischter Text

Hier ist ein deutscher Text mit eingebetteten English phrases und auch quelques mots français zur Demonstration der Spracherkennung innerhalb eines Textes.

Hinweis: Die korrekte Aussprache hängt von den verfügbaren Stimmen im Browser ab. Nicht alle Browser stellen für alle Sprachen Stimmen bereit.
